#### HTML 5 有哪些新标签？

HTML 5 引入了很多新的标签，一些常见的包括：

1. **&lt;header&gt;**: 用于定义文档或区域的页眉。
1. **&lt;nav&gt;**: 用于定义文档的导航链接部分。
1. **&lt;main&gt;**: 用于定义文档的主要内容部分。
1. **&lt;section&gt;**: 用于定义文档中的节（section）。
1. **&lt;article&gt;**: 用于定义独立的自包含文章。
1. **&lt;aside&gt;**: 用于定义与文档内容相关的侧边栏内容。
1. **&lt;footer&gt;**: 用于定义文档或区域的页脚。
1. **&lt;video&gt;**: 用于在网页中嵌入视频。
1. **&lt;audio&gt;**: 用于在网页中嵌入音频。
1. **&lt;canvas&gt;**: 用于在网页中绘制图形。
1. **&lt;progress&gt;**: 用于表示任务的完成进度。
1. **&lt;meter&gt;**: 用于表示度量值，如温度或尺寸。
1. **&lt;time&gt;**: 用于表示日期或时间。
1. **&lt;datalist&gt;**: 用于定义输入框的选项列表。
1. **&lt;output&gt;**: 用于定义计算结果输出的容器。

等等。这些标签的引入使得 HTML 的语义化更加丰富，同时也方便了搜索引擎和屏幕阅读器等工具的理解。

#### HTML5 对比 HTML4 有哪些不同之处？HTML5 有哪些新特性？

HTML5 相对于 HTML4 来说有以下不同之处：

1. **结构和元素方面**：HTML5 新增了一些新的结构化元素，如 header、footer、article、section、nav、aside、main 等，这些元素能够让开发者更好地划分文档结构，同时也更有利于 SEO。
1. **多媒体方面**：HTML5 对多媒体方面的支持更加完善，增加了 audio 和 video 元素，支持原生的多媒体播放、控制等功能。同时，HTML5 也提供了更好的支持媒体流的技术。
1. **表单方面**：HTML5 新增了很多新的表单元素和属性，如 date、time、email、url、range、number、search、required 等等，使得表单元素更加丰富、便于开发者处理表单数据。
1. **语义化方面**：HTML5 强调语义化，通过使用新增的语义化元素和属性，开发者可以更加准确地描述文档内容和结构。
1. **存储和缓存方面**：HTML5 提供了 Web Storage 和 Web SQL 数据库两种方式用于客户端存储和缓存数据。
1. **性能方面**：HTML5 提供了一些新的 API，如 Web Workers、Web Sockets、IndexedDB 等，用于提高 Web 应用的性能和响应速度。

HTML5 的新特性包括但不限于：

1. **新的结构化元素**：header、footer、article、section、nav、aside、main 等。
1. **多媒体元素**：audio、video、source 等。
1. **新的表单控件和属性**：date、time、email、url、range、number、search、required 等。
1. **语义化标签**：article、section、nav、header、footer、aside 等。
1. **存储和缓存**：Web Storage、Web SQL 数据库、Application Cache 等。
1. **Web Workers**：允许开发者创建一个后台进程，以提高 Web 应用的性能和响应速度。
1. **Web Sockets**：允许开发者建立基于 TCP 的双向通信，实时通讯。
1. **Canvas 和 SVG**：用于图形和动画的绘制。
1. **Geolocation API**：用于获取用户地理位置信息。

#### 怎么理解语义化标签？用过哪些语义化标签？

语义化标签是指在 HTML 中使用具有特定含义的标签，以便更好地描述文档的结构和内容，使得文档更易于理解、维护和访问。语义化标签不仅可以提高页面的可读性和可访问性，而且对搜索引擎优化(SEO)也有很大的帮助。

常见的语义化标签有：

1. **&lt;header&gt;**：定义文档或节的页眉。
1. **&lt;nav&gt;**：定义导航链接的部分。
1. **&lt;main&gt;**：定义文档的主要内容。
1. **&lt;article&gt;**：定义独立的自包含文章或内容块。
1. **&lt;aside&gt;**：定义页面或文章的侧边栏。
1. **&lt;section&gt;**：定义页面或文章的某个章节或区域。
1. **&lt;footer&gt;**：定义文档或节的页脚。
1. **&lt;figure&gt;**：定义一段独立的流内容，如一张图片、一段视频等。
1. **&lt;figcaption&gt;**：定义 &lt;figure&gt; 元素的标题或说明。

使用语义化标签可以让代码更加清晰易懂，并且能够使搜索引擎更好地理解页面的结构和内容，提高网页的排名和曝光度。同时也可以让页面在无样式情况下有更好的可读性和可访问性，提高用户体验。

#### src 和 href 的区别？

相同点：都是用来引入外部资源路径。

不同点：

1. src 是用于指定引入资源的路径，比如 &lt;img src="example.jpg"&gt; 表示引入一张图片。
1. href 则是用于指定链接的 URL，比如 &lt;a href="https://example.com"&gt; 表示该链接指向 https://example.com。

#### script 引用时 defer 和 async 的区别是什么？

defer 和 async 都是用于异步加载脚本，不过两者的区别在于执行的时机。

1. defer 会在文档解析完毕后才会执行，
2. async 则是在下载完成后立即执行。

这也意味着如果有多个 defer 脚本，它们的执行顺序是按照它们在文档中出现的顺序执行；而对于多个 async 脚本，则不能保证它们的执行顺序。

#### img 的 srcset 属性的作用？

srcset 是一个 HTML 标签中用于指定图片的属性，用于为不同大小的显示设备提供不同分辨率的图片。其作用是让浏览器根据屏幕大小和显示设备的分辨率选择合适的图片，以提高网页的性能和用户体验。

srcset 属性可以指定多个图片资源及其对应的设备像素比（DPR），浏览器可以根据当前设备的 DPR 和屏幕宽度选择合适的图片。例如：

```
<img src="default.jpg"
     srcset="small.jpg 480w,
             medium.jpg 768w,
             large.jpg 1280w"
     alt="Alternative text">
```

在上面的代码中，srcset 属性指定了三个图片资源，分别适用于屏幕宽度为 480、768 和 1280 像素的设备。浏览器会根据当前设备的宽度和 DPR 自动选择合适的图片，如果当前设备的 DPR 是 1，则选择宽度为 480 像素的 small.jpg 图片，如果 DPR 是 2，则选择宽度为 768 像素的 medium.jpg 图片。

需要注意的是，如果浏览器不支持 srcset 属性，会使用 src 属性指定的默认图片。可以通过 sizes 属性指定图片在不同宽度下的显示方式，例如：

```
<img src="default.jpg"
     srcset="small.jpg 480w,
             medium.jpg 768w,
             large.jpg 1280w"
     sizes="(max-width: 480px) 100vw, 50vw"
     alt="Alternative text">
```

在上面的代码中，sizes 属性指定了图片在不同宽度下的显示方式，如果屏幕宽度小于等于 480 像素，则显示整个图片，否则显示图片宽度的一半。

#### 响应式图片处理优化：Picture 标签？

响应式图片处理是指在不同的屏幕尺寸和设备上为图像提供最佳的显示效果。使用 picture 标签是响应式图片处理的一种常见方法，它可以让网页根据设备的尺寸和分辨率自动选择最合适的图像大小和格式。

picture 标签允许开发者在同一个文档内包含多个不同版本的图片，每个版本可以针对不同的屏幕尺寸和像素密度进行优化。在 picture 标签中，开发者可以使用 source 元素来指定不同版本的图片，并使用 media 属性来指定何时应该显示这些图片。例如：

```
<picture>
  <source media="(max-width: 768px)" srcset="small.jpg">
  <source media="(min-width: 769px)" srcset="large.jpg">
  <img src="fallback.jpg" alt="My Image">
</picture>
```

在上面的例子中，如果屏幕宽度小于等于 768 像素，浏览器将使用 small.jpg。如果屏幕宽度大于 768 像素，则浏览器将使用 large.jpg。如果浏览器无法加载任何一个 source 元素指定的图像，它将使用 img 元素的 src 属性指定的图像作为备用方案。

使用 picture 标签进行响应式图片处理有以下优点：

1. 显示的图像可以根据设备的尺寸和像素密度进行优化，从而提高页面的性能和用户体验。
1. 开发者可以为不同的屏幕尺寸和设备提供不同的图像版本，以获得最佳的视觉效果。
1. 支持多个 source 元素，可以针对不同的条件提供多个备选方案。
1. 适用于各种图像格式，包括 JPEG、PNG、WebP 等。

总之，使用 picture 标签可以为网站的图像提供更好的响应式处理，并提高用户体验。

#### Canvas 和 SVG 的区别是什么？

Canvas 和 SVG 都是用于创建图形的 Web 技术，它们的主要区别在于实现方式和应用场景。

1. **实现方式**：Canvas 使用 JavaScript API 绘制位图，而 SVG 则使用 XML 标记绘制矢量图形。
1. **渲染方式**：Canvas 渲染的是像素，而 SVG 渲染的是矢量图形。
1. **适用场景**：Canvas 更适合用于创建动态和复杂的图像，如游戏和数据可视化等。SVG 则更适合用于创建静态和基于文本的图像，如图标、标识和矢量图等。
1. **缩放和变换**：由于 Canvas 绘制的是位图，因此它在缩放和变换方面的效果不如 SVG。SVG 可以无损缩放和变换，并且可以轻松实现动画和交互效果。
1. **性能**：Canvas 相对于 SVG 具有更好的性能，因为它使用的是位图，对于大型和复杂的图形绘制，Canvas 的渲染速度更快。

综上所述，Canvas 和 SVG 的主要区别在于它们的实现方式、渲染方式、适用场景、缩放和变换、性能等方面。开发者需要根据具体的需求和应用场景选择合适的技术来创建图形。

#### 行内元素有哪些？块级元素有哪些？空元素有哪些？

在 HTML 中，元素可以分为三类：行内元素、块级元素和空元素。它们的主要区别在于显示方式、默认宽度和包含内容等方面。

行内元素：行内元素在文本流中显示，一般不会独占一行，仅占据其内容的宽度。常见的行内元素包括 &lt;a&gt;、&lt;span&gt;、&lt;em&gt;、&lt;strong&gt;、&lt;code&gt;、&lt;img&gt;、&lt;input&gt; 等。

块级元素：块级元素在文本流中独占一行，一般占据其父元素的全部宽度。常见的块级元素包括 &lt;div&gt;、&lt;p&gt;、&lt;h1&gt;~&lt;h6&gt;、&lt;ul&gt;、&lt;ol&gt;、&lt;li&gt;、&lt;table&gt; 等。

空元素：空元素是指没有内容的元素，也称为自闭合标签或单标签。常见的空元素包括 &lt;br&gt;、&lt;hr&gt;、&lt;img&gt;、&lt;input&gt;、&lt;meta&gt;、&lt;link&gt; 等。

需要注意的是，HTML5 中新增了一些元素，如 &lt;article&gt;、&lt;section&gt;、&lt;header&gt;、&lt;footer&gt; 等，它们的显示方式和默认样式与块级元素类似，但它们更具有语义化的意义，因此也可以被归为块级元素的一种。

总的来说，行内元素、块级元素和空元素是 HTML 中常见的元素类型，了解它们的特点和使用场景对于编写符合标准的 HTML 文档是非常重要的。

#### title 与 h1 的区别、b 与 strong 的区别、i 与 em 的区别？

在 HTML 中，title、h1、b、strong、i、em 等标签都是用来标记文本的元素，它们之间的区别在于语义和样式的不同。

title 与 h1 的区别：

1. title 标签是 HTML 文档的元数据之一，用于定义网页的标题，一般在 <head> 中使用。
1. h1 标签用于表示页面的主标题，一般作为页面结构的顶层元素，每个页面只应出现一次。

b 与 strong 的区别：

1. b 标签用于将文本加粗显示，通常用于强调一些关键字或短语，但不表示其语义上的强调。
1. strong 标签用于表示文本的重要性或紧急程度，通常也会将文本加粗显示，但同时也具有语义上的强调作用。

i 与 em 的区别：

1. i 标签用于将文本以斜体字显示，通常用于表示专业术语、外文短语等，但不表示其语义上的强调。
1. em 标签用于表示文本的强调或重点，通常也会以斜体字显示，但同时也具有语义上的强调作用。

#### 说一下 HTML5 的离线存储？

HTML5 的离线存储（Offline Web Applications）是一种利用浏览器本地缓存存储 Web 应用程序资源的技术，使得用户在离线状态下仍然可以访问已经访问过的页面，提高用户体验。

离线存储的实现步骤如下：

1. 在 HTML 文件中添加 manifest 属性指向一个描述离线应用的 cache.manifest 文件。
1. 在 cache.manifest 文件中列出需要缓存的文件，包括 HTML 文件、CSS 文件、JavaScript 文件、图像文件等。
1. 在服务器端设置 mime-type 为 text/cache-manifest。
1. 当用户访问离线页面时，浏览器会自动下载并缓存这些文件，并在离线状态下自动从缓存中读取文件。

离线存储的优点是可以减少网络请求，加快页面的加载速度，提高用户体验；同时也可以提高 Web 应用的可靠性，即使在网络不稳定或无网络的情况下，用户也能够访问页面。但是需要注意的是，离线存储可能会导致缓存文件的更新问题，因此需要在更新文件时及时更新 cache.manifest 文件，以保证用户获得最新的文件。

#### html 常见兼容性问题？

HTML 的兼容性问题主要体现在不同浏览器的解析方式和支持程度不同，导致页面在不同浏览器中显示效果不同或者出现错误。以下是一些常见的 HTML 兼容性问题：

1. **盒子模型**：不同浏览器对盒子模型的解析方式不同，例如 IE6 及以下版本将元素的边框和内边距计算在宽度之内，而标准的盒子模型是将边框和内边距计算在盒子外部的。
1. **样式表**：不同浏览器对 CSS 的支持程度不同，某些属性或属性值在某些浏览器中可能无法正常显示或支持，例如 CSS3 的一些新属性。
1. **表单元素**：某些浏览器可能对 HTML5 的表单元素不支持，例如 IE9 及以下版本对 HTML5 的表单元素支持不完全。
1. **图片**：某些浏览器可能无法显示某些图片格式，例如 WebP 格式的图片在某些浏览器中可能无法正常显示。
1. **标签**：某些标签在某些浏览器中可能无法正常解析，例如 IE6 及以下版本对 HTML5 标签的支持不完全。

为了避免 HTML 兼容性问题，可以使用一些通用的解决方案，例如：

1. 使用标准的 HTML 代码和 CSS 样式，并尽量遵循 W3C 标准。
1. 针对不同浏览器使用不同的 CSS hack 或 JavaScript shim 解决兼容性问题。
1. 使用 polyfill 库来实现 HTML5 新特性的兼容性，例如 Modernizr。
1. 使用 CSS Reset 或 Normalize.css 等库来消除浏览器间的样式差异。

#### 说说 img 标签的 onerror 事件，图片加载失败的处理方法？

在 HTML 中，img 标签用于显示图片，而 onerror 事件是 img 标签的一个事件属性，它会在图片加载失败时触发。具体来说，当图片的 URL 不可用或图片文件损坏等情况导致图片无法正常加载时，会触发 onerror 事件。

可以利用 onerror 事件来处理图片加载失败的情况，以下是一些处理方法：

1. **使用备用图片**：可以在 onerror 事件中替换为另一张备用图片，以保证用户能够看到一个有效的图片。
1. **显示错误信息**：可以在 onerror 事件中显示错误信息，以告知用户图片无法加载的原因。
1. **重试加载**：可以在 onerror 事件中尝试重新加载图片，以提高加载成功率。
1. **删除无效图片**：可以在 onerror 事件中删除无效图片，以避免无效图片对页面性能的影响。

以下是一个使用 onerror 事件处理图片加载失败的示例代码：

```
<img src="image.png" onerror="this.onerror=null; this.src='backup.png';">
```

在该示例中，当原始图片加载失败时，会触发 onerror 事件，然后将图片的 URL 替换为备用图片的 URL，以确保用户能够看到有效的图片。注意，需要在 onerror 事件中将该事件的处理程序设为 null，以避免出现无限递归的情况。

#### 前端页面由哪三层构成？

前端页面通常由以下三层构成：

1. **结构层（HTML）**：结构层负责页面的基本结构和内容的展示。HTML（HyperText Markup Language）是用于定义 Web 页面结构的标记语言，它提供了丰富的标签和属性，用于描述页面的结构和内容。
1. **表示层（CSS）**：表示层负责页面的样式和布局。CSS（Cascading Style Sheets）是一种用于描述文档样式和布局的样式表语言，它可以控制页面元素的外观、排列和位置等属性。
1. **行为层（JavaScript）**：行为层负责页面的交互和动态效果。JavaScript 是一种脚本语言，可以在页面中添加交互和动态效果，例如表单验证、页面切换和动画效果等。

这三层之间相互独立，但是又相互关联，通过它们的协同工作，可以创建出丰富、美观、功能强大的前端页面。

#### 说说你对 WEB 标准以及 W3C 的理解与认识？

**WEB 标准**是指为 Web 技术的开发和应用制定的一系列标准规范，包括 HTML、CSS、JavaScript 等技术的规范，它的目的是为了提高 Web 应用的可访问性、可用性和可维护性。WEB 标准的实现可以提高网站在不同浏览器和设备上的兼容性，使得用户在使用网站时能够获得更好的体验。

**W3C（World Wide Web Consortium）**是一个负责制定和推广 Web 标准的组织，它由互联网的发明者 Tim Berners-Lee 创立，致力于推动 Web 技术的发展和标准化。W3C 的使命是为 Web 的长期发展制定开放的标准，同时提供 Web 技术的测试工具和支持服务，推广 Web 技术的发展和普及，以确保 Web 技术的互通性和可持续性发展。

WEB 标准和 W3C 是相辅相成的，WEB 标准是基于实践总结出来的规范，而 W3C 则是负责制定和推广这些规范的组织。WEB 标准的制定需要经过广泛的讨论和实践验证，而 W3C 则要确保这些标准的质量和互通性。WEB 标准和 W3C 对于推动 Web 技术的发展和推广具有重要的作用，它们的存在有助于保证 Web 技术的发展和应用的可持续性。

#### 为什么最好把 CSS 的 &lt;link&gt; 标签放在 &lt;head&gt; &lt;/head&gt; 之间？

把 CSS 的 &lt;link&gt; 标签放在 &lt;head&gt; 标签中的好处有以下几点：

1. **加载顺序**：将 CSS 的 &lt;link&gt; 标签放在 &lt;head&gt; 标签中，可以使浏览器在渲染页面之前就加载 CSS 文件，以保证页面渲染时样式已经生效，避免了样式闪烁或加载延迟的问题。
1. **防止 FOUC**：当 CSS 样式文件延迟加载时，浏览器会先渲染 HTML 页面的结构和内容，再通过 JavaScript 动态加载 CSS 样式文件。这会导致页面一开始是没有样式的，会出现一瞬间的白屏或者样式错乱的情况，这就是 FOUC (Flash of Unstyled Content)。将 CSS 的 &lt;link&gt; 标签放在 &lt;head&gt; 标签中可以有效避免这个问题。
1. **提高渲染效率**：如果将 CSS 的 &lt;link&gt; 标签放在 &lt;body&gt; 标签中，那么浏览器会在渲染 HTML 页面的同时加载 CSS 样式文件，这会导致页面渲染速度变慢，用户体验变差。将 CSS 的 &lt;link&gt; 标签放在 &lt;head&gt; 标签中可以保证浏览器在渲染页面之前就加载 CSS 样式文件，提高了页面渲染效率。

综上所述，把 CSS 的 &lt;link&gt; 标签放在 &lt;head&gt; 标签中是最好的选择，可以保证页面渲染时样式已经生效，提高了页面渲染效率，避免了样式闪烁或加载延迟的问题，同时也避免了 FOUC 的出现。

#### 为什么最好把 JS 的 &lt;script&gt; 标签恰好放在 &lt;/body&gt; 之前？

把 JS 的 &lt;script> 标签放在 &lt;/body&gt; 之前的好处有以下几点：

1. **加载顺序**：将 JS 的 &lt;script&gt; 标签放在 &lt;/body&gt; 标签之前，可以保证 HTML 页面的结构和内容已经全部渲染完成后才加载 JS 文件，避免了在 JS 文件加载时出现页面结构和内容不完整的情况。
1. **提高页面加载速度**：将 JS 的 &lt;script&gt; 标签放在 &lt;/body&gt; 标签之前，可以避免阻塞页面的渲染，因为 JS 文件的加载和解析是会阻塞浏览器渲染 HTML 页面的，特别是当 JS 文件较大时，更容易影响页面加载速度，降低用户的体验感。
1. **优化浏览器缓存**：将 JS 的 &lt;script&gt; 标签放在 &lt;/body&gt; 标签之前，可以利用浏览器的缓存机制，当用户再次访问相同页面时，浏览器可以直接从缓存中加载 JS 文件，减少了网络请求和服务器的负载，提高了页面的响应速度。

综上所述，将 JS 的 &lt;script&gt; 标签放在 &lt;/body&gt; 标签之前是最好的选择，可以避免阻塞页面的渲染，提高页面的加载速度和响应速度，同时也利用了浏览器的缓存机制，优化了用户的体验感。
